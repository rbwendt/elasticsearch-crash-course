The interface

ES works through a RESTish JSON API.

The interface

POST for creating data. E.g. saving documents.

The interface

GET for retrieving data. E.g. searching for and loading documents.

The interface

POST for updating data. E.g. changing existing documents.

The interface

DELETE for deleting data. E.g. deleting documents.

types of queries

types of queries

match_all

matches everything

types of queries

match

Finds matches in a given field. Goto for full text search

types of queries

match_phrase

Finds matches in a given field that are near each other

types of queries

multi_match

Like match but for multiple fields.

types of queries

common_terms

A match query that doesn't discount common words. E.g. "not" is a stop word, but "not happy" and "happy" have different meanings.

types of queries

query_string

Query using lucene query string syntax.

e.g. "query" : "this AND that OR thus"

types of queries

simple_query_string

A simpler, more robust version of the query_string syntax suitable for exposing directly to users.

types of queries

term

An exact match query

types of queries

terms

Return documents that match any of the given terms in the specified field

types of queries

range

Return documents where the specified field is in the specified range.

types of queries

exists

Find documents where the field specified contains any non-null value.

types of queries

missing

Find documents where the field specified contains a null value or is not set. Opposite of exists.

types of queries

regexp

Find documents where a given field matches the regex in the query.

types of queries

fuzzy

Find documents where a given field is similar to the query. Uses Levenshtein

types of queries

type

Find documents of the given type.

types of queries

ids

Find documents with the given ids.

compound queries

Operate on the results of other queries.

E.g. bool query can say match either this query or that query.

geo queries

For querying geo data.

Supports geo_shape, geo_bounding_box, geo_distance, geo_distance_range, geo_polygon, geohash_cell .

mapping

ES acts like a schemaless database, but it isn't. It auto- generates a schema called a mapping.

mapping

ES will automatically generate a schema for your documents, but it's better to explicitly state one. The generated mapping can be wrong.

mapping

The mapping API is fairly simple. Just supply a list of fields and their types in JSON format.

This is where you specify what analysis you want done on your fields.

Aggregations

For building summaries of data.

Metrics ggregations

Has aggregations like min, max, avg, percentiles, etc.

Bucket ggregations

Makes buckets of documents. E.g. all shoes that are size 9.

Pipeline ggregations

Works with the results of other aggregations. E.g. go from position to get velocity, or vice versa.
